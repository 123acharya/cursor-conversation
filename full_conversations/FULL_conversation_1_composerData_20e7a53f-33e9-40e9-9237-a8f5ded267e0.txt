## Summary of what I understand

### 1. Stock Entry — structure and purpose

Purpose: Core document for inventory movements between warehouses.

Structure:
- Main fields: `stock_entry_type` (determines route), `from_warehouse`, `to_warehouse`, `posting_date`
- Items table: `item_code`, `qty`, `s_warehouse`, `t_warehouse`, `serial_no`
- Custom PCB table: Appears when item `PA01 0001` is entered

How it works:
- Stock Entry Type defines golden routes (enforced server-side)
- Routes validated on submit (cannot bypass)
- PCB table integrated for serial number tracking
- After submit: Updates PCB Master and creates PCB Ledger entries

---

### 2. PCB IQC Result — structure and purpose

Purpose: Entry UI form for operators to enter PCB test results.

Structure:
- Main form: `pcb` (Item link), `pcb_scan` (barcode input), `iqc_date`
- Child table (PCB IQC Line): `pcb_serial_no`, `pts1_status`, `pts2_status`, `pts3_status`, `overall_status`, `dominant_channel`, `max_value`, `remarks`

How it works:
- Operators enter test results (single or batch)
- Data saved to PCB IQC Line (background table)
- Server script updates PCB Master and creates PCB Ledger entry
- Can search by serial number to update existing results

---

### 3. PCB Ledger — structure and purpose

Purpose: Complete audit trail — transaction log for every PCB event.

Structure:
- Links: `pcb_serial_no` → PCB Master
- Event tracking: `event_type`, `event_time`, `user`, `ref_doctype`, `ref_docname`
- Status tracking: `overall_status`, `prev_overall_status`, `new_overall_status`
- Movement tracking: `from_warehouse`, `to_warehouse`
- Test data: `pts1_status`, `pts2_status`, `pts3_status`, `remarks`

How it works:
- Created automatically by server scripts
- One entry per event (test, movement, status change)
- Complete history queryable by serial number
- Event types: PTS_TEST, IQC_PASS, IQC_FAIL, MOVED_TO_STORE, etc.

---

### 4. PCB Master — structure and purpose

Purpose: Single source of truth — current state of each PCB.

Structure:
- Primary key: `pcb_serial_no` (serial number)
- Status: `overall_status` (NEW, PTS_PENDING, PASS, FAIL, etc.)
- Test results: `pts1_status`, `pts2_status`, `pts3_status`
- Location: `last_warehouse` (current warehouse)
- References: `last_event_ref_doctype`, `last_event_ref_docname`
- Test data: `dominant_channel`, `max_value`, `iqc_date`, `last_tested_on`

How it works:
- Created/updated automatically when PCB IQC Result saved or Stock Entry submitted
- Always reflects current state (status, warehouse, test results)
- Links to last event (Stock Entry or PCB IQC Result)
- Queryable by serial number for current status

---

### How they work together

```
PCB IQC Result (Entry Form)
    ↓ saves to
PCB IQC Line (Background Table) ← Single Source of Truth
    ↓ updates
PCB Master (Current State)
    ↓ creates
PCB Ledger (Audit Trail)

Stock Entry (Movement)
    ↓ validates route
    ↓ updates stock
    ↓ updates
PCB Master (Current State)
    ↓ creates
PCB Ledger (Audit Trail)
```

## Quick fix (recommended)

### Manual fix via ERPNext UI (about 5 minutes)

1. Login to ERPNext: http://localhost:8080
2. Go to: Customize → Client Script → List
3. Find and open: `Stock Entry Purpose + Pcb Scan Script`
4. Add this code at the beginning of the script:

```javascript
// Auto-set Default Target Warehouse for Rejected from Assembly
frappe.ui.form.on('Stock Entry', {
    stock_entry_type(frm) {
        if (frm.doc.stock_entry_type === 'Rejected from Assembly') {
            if (!frm.doc.to_warehouse || frm.doc.to_warehouse !== 'Rejected items - R') {
                frm.set_value('to_warehouse', 'Rejected items - R');
            }
        }
    },
    
    onload(frm) {
        if (frm.doc.stock_entry_type === 'Rejected from Assembly') {
            if (!frm.doc.to_warehouse || frm.doc.to_warehouse !== 'Rejected items - R') {
                frm.set_value('to_warehouse', 'Rejected items - R');
            }
        }
    },
    
    refresh(frm) {
        if (frm.doc.stock_entry_type === 'Rejected from Assembly') {
            if (!frm.doc.to_warehouse || frm.doc.to_warehouse !== 'Rejected items - R') {
                frm.set_value('to_warehouse', 'Rejected items - R');
            }
        }
    }
});
```

5. Save the script
6. Clear cache: Settings → Clear Cache
7. Test: Create a new Stock Entry with type "Rejected from Assembly"

## What this does

- Auto-sets Default Target Warehouse = "Rejected items - R" when:
  - Stock Entry Type is set to "Rejected from Assembly"
  - Form loads with that type
  - Form refreshes
- Prevents the validation error by ensuring the field is always set
